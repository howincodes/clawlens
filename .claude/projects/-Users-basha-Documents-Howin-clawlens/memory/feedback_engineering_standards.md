name: Engineering standards
description: User requires professional, production-quality code with no hacks or workarounds. Product is open source.
type: feedback
---

Always fix the product properly — no hacks, no workarounds, no "good enough" patches.

**Why:** ClawLens is an open-source product. Code quality must be professional. The user has been frustrated by patching on patches in the past (v0.1).

**How to apply:** Every fix must address the root cause. If something is broken, understand why before fixing. Never add a workaround when a proper fix exists. Write code that an open-source contributor would be proud to read.
